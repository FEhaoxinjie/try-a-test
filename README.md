#### 支持日期xx 或者 月份xx
  - 对于月份xx
    + 先判断是不是31号（这里加了一个随机数，如果大于0.5就执行后面的操作。如果小于直接默认为1月31号，查询得知为什么很少有人在31号，说法是如果是农村户口习惯用农历做生日，但是农历没有31号所以一部分人会把出生在公历31号的生日改到农历生日，所以这里就先断定如果是农村户口而且是31号，那么就认为是不好改变或者无法改变，那么只能是1月31号，如果改到农历那么年份可能会少1年；为什么是1：1以为第六次人口普查，城市和农村比例大约为1：1），如果是就取出1，3，5，7，8，10，12月份的后一个星座（每个月份有两个星座，第一个月份持续时间较长，第二个较短）；
    + 如果是其他日期，那么就用这个日期减去当前月份第一个星座的持续时间，如果小于那么就属于前一个如果不小于就属于后一个星座，然后返回可能的星座列表

  - 对于日期是xx；

	就取出当前月份的前一个星座，返回


  - 对于日期和月份都存在的直接返回确定星座





### 对于如果是真实身份证，是否有唯一解

  - 如果为真实身份证号，那么翻阅公民身份证规定第15位到第17位为顺序号 即同一地址码下，同年同月同日出生的人的顺序号，第1位到第6位是地址码，如果能得到当地所有的出生人口记录，排除没有31号的月份找到其余31号出生人数大于等于251人的月份可以极大缩小范围，但是仍无法有唯一解，此外如果92年时大多数农村生育不会在医院登记都是出生以后一段时间才会办户口，也会影响结果。
